Dikkat Mekanizması
Query, Key, Value
Her token Q, K, V vektörlerine dönüştürülür. Dikkat skoru hesaplanır:
Attention(Q,K,V) = softmax(QKᵀ / √d_k) × V
Q ile tüm K’ların nokta çarpımı alınır, √d_k ile normalize edilir, softmax uygulanır, V ile ağırlıklı toplam alınır.

06 — Ders Notları

Computer Networks

Kurose & Ross — A Top-Down Approach, 8th Edition · İlk 5 Bölüm

1.1 — İnternet Nedir?
Nuts-and-Bolts Tanım & Protokol

Host (End System) — İnternet’e bağlı tüm cihazlar: masaüstü bilgisayarlar, akıllı telefonlar, tabletler, TV’ler, termostatlar, arabalar. 2017 itibarıyla ~18 milyar, 2022’ye kadar 28.5 milyara ulaşacağı tahmin edilmektedir.

Packet Switch — Gelen bir paketi, bir giden bağlantıya yönlendiren ağ düğümü. İki ana tipi vardır: Router (ağ çekirdeğinde) ve Link-layer switch (erişim ağlarında).

Communication Link — Paket geçişini sağlayan fiziksel ortam (koaksiyel kablo, bakır tel, optik fiber, radyo spektrumu). Her bağlantının bant genişliği (transmission rate) bit/saniye cinsinden ölçülür.

Packet — Gönderici end system’in verisini parçalara ayırıp her parçaya başlık byte’ları ekleyerek oluşturduğu bilgi paketi. Hedef end system’e ulaştığında yeniden birleştirilir.

Route / Path — Paketin gönderici end system’den alıcı end system’e kadar geçtiği bağlantı ve anahtar dizisi.

ISP (Internet Service Provider) — End system’lerin İnternet’e erişimini sağlayan kurum. Her ISP kendi içinde paket anahtarları ve bağlantılardan oluşan bir ağdır. Alt-tier ISP’ler, üst-tier (national/international) ISP’ler aracılığıyla birbirine bağlanır.

TCP/IP — Transmission Control Protocol / Internet Protocol
İnternet’in temel protokol ikili. IP, router’lar ve end system’ler arasında gönderilen paketlerin formatını belirler.

RFC (Request for Comments) — IETF (Internet Engineering Task Force) tarafından yayımlanan İnternet standart belgeleri. TCP, IP, HTTP, SMTP gibi protokolleri tanımlar. Günümüzde ~9000 RFC mevcuttur.

Socket Interface — Bir end system üzerinde çalışan programın, İnternet altyapısını kullanarak başka bir end system’deki programa veri iletmesini sağlayan kural seti (API).

İnternet’in Kavramsal Yapısı
Uygulamalar (Web, E-posta, Video, DNS…)
İnternet Protokolleri (TCP/IP)
Ağ Altyapısı (Router’lar, Anahtarlar)
Erişim Ağları (DSL, Kablo, WiFi, Fiber)
Fiziksel Ortam (Kablo, Fiber, Radyo)
1.2 — Ağ Kenarı
Erişim Ağları & Fiziksel Ortam

Access Network — End system’i, İnternet yolundaki ilk router’a (edge router) bağlayan ağ. Üç ana kategori vardır: ev erişim ağları, kurumsal erişim ağları ve geniş alan kablosuz erişim.

DSL (Digital Subscriber Line) — Telefon şirketlerinin mevcut bakır telefon hatlarını kullanarak sağladığı genişbant erişim. DSLAM (DSL Access Multiplexer) telco merkezindedir. Tipik hız: 24 Mbps indirme, 2.5 Mbps yükleme.

HFC (Hybrid Fiber Coax) — Kablo TV şirketlerinin fiber ve koaksiyel kablo karışımıyla sunduğu erişim. Frekans bölmeli çogullama (FDM) ile kullanılır. CMTS (Cable Modem Termination System) ile merkeze bağlanır. Önemli: DSL’den farklı olarak HFC paylaşımlıdır — komşular bant genişliğini paylaşır.

FTTH (Fiber to the Home) — Fiber optik kablo doğrudan eve getirilir. AON (Active Optical Network) ve PON (Passive Optical Network) olmak üzere iki tipi vardır. Gigabit hızlarına ulaşabilir.

Ethernet — Büro/kampüs erişiminde en yaygın teknoloji. Twisted-pair bakır kablo ile Ethernet switch’e bağlanılır. Tipik hızlar: 100 Mbps, 1 Gbps, 10 Gbps.

WiFi (802.11) — Kablosuz LAN. Access point (AP) aracılığıyla bağlanılır. 54 Mbps’dan yüzlerce Mbps’e kadar çeşitli standardlar mevcut.

3G / LTE 4G / 5G — Geniş alan mobil kablosuz erişim. Akıllı telefonlar baz istasyonları (cell towers) üzerinden internete bağlanır.

Fiziksel Ortam Türleri:
• Guided media: Twisted-pair bakır (UTP/STP) — En ucuz, telefon/Ethernet
• Coaxial cable — Çift iletken, geniş bant paylaşımlı ortam
• Fiber optic — Işık darbeleri, 10–100 Gbps, elektromanyetik girişime karşı bağışık
• Unguided media: Karasal radyo, uydu, 3G/4G/5G
1.3 — Ağ Çekirdeği
Paket Anahtarlama vs Devre Anahtarlama

Packet Switching (Paket Anahtarlama) — Mesaj küçük paketlere bölünür. Her paket bağımsız olarak yönlendirilir. Store-and-forward prensibi: Router paketin tümünü almadan iletmeye başlamaz.

Store-and-Forward Gecikmesi:
L = paket boyutu (bit), R = bağlantı kapasitesi (bps)
1 paketin 1 bağlantıdan geçiş süresi = L/R saniye
N bağlantı ve P paket: toplam = (N + P - 1) × L/R

Queuing Delay & Packet Loss — Paketler çok hızlı gelirse router’ın çıkış kuyruğu (output queue/buffer) dolar. Dolup taşınca paket kaybı oluşur.

Forwarding Table — Her router, hedef adresi inceler ve forwarding table’ından hangi çıkış bağlantısına yönlendireceğini bulur. Bu tablo routing protokolleri tarafından otomatik doldurulur.

Circuit Switching (Devre Anahtarlama) — Kaynak ile hedef arasında özel bir devre rezerve edilir (örn. geleneksel PSTN telefon ağları). Kaynaklar iletim olmasa dahi bloke edilir. İki multiplexing yöntemi:

FDM (Frekans Bölmeli Çoğullama): Her bağlantıya ayrı frekans bandı
TDM (Zaman Bölmeli Çoğullama): Her bağlantıya döngüsel zaman dilimi (slot)

Paket vs Devre Anahtarlama Karşılaştırması: Paket anahtarlama kaynak kullanımını optimize eder. Birden fazla kullanıcı için aynı kapasiteyle çok daha fazla kullanıcı destekler (statistical multiplexing). Ama tıkanıklık garantisi vermez; devre anahtarlama verir.

Network of Networks — İnternet ISP'lerin hiyerarşik ağıdır. Access ISP → Regional/National ISP → Tier-1 ISP. IXP (Internet Exchange Point)'ler ISP'lerin birbirine bağlandığı noktalardır. Google, Microsoft, Akamai gibi büyük içerik sağlayıcılar kendi ağlarını işletir.

1.4 — Gecikme, Kayıp ve Throughput
Paket Anahtamalı Ağlarda Performans Analizi

Nodal Processing Delay (dproc) — Router'ın paketin başlığını inceleyip doğru çıkış bağlantısını belirlemesi için geçen süre. Genellikle mikrosaniye mertebesinde. Bit hataları da kontrol edilir.

Queuing Delay (dqueue) — Paketin çıkış kuyruğunda beklemesi için geçen süre. Değişkendir: kuyruk boşsa sıfır, doluysa saniyeler sürebilir. En önemli ve öngörülemeyen gecikme.

Transmission Delay (dtrans) — Paketin tüm bitlerini bağlantıya aktarmak için gereken süre. dtrans = L/R (L: paket boyutu bit, R: bağlantı kapasitesi bps).

Propagation Delay (dprop) — Bir bitin bağlantı üzerinde fiziksel olarak yayılma süresi. dprop = d/s (d: bağlantı uzunluğu, s: yayılma hızı ≈ 2×108 m/s).

Toplam Nodal Gecikme:
dnodal = dproc + dqueue + dtrans + dprop

Trafik Yoğunluğu (Traffic Intensity): a = paket varış hızı (pkt/sn)
I = L·a / R (0 ≤ I < 1 olmak zorunda, I → 1 ise kuyruk → ∞)

Packet Loss — Kuyruk (buffer) tamamen dolduğunda yeni gelen paket düşürülür. Ağ katmanı kaybı tespit etmez; güvenilir iletim gerekiyorsa üst katman (TCP) yeniden iletmek zorundadır.

Throughput (Verim) — Birim zamanda gerçekte iletilen veri miktarı (bps). Instantaneous (anlık) ve average (ortalama) throughput olmak üzere iki türü vardır.

Bottleneck Link (Darboğaz Bağlantısı):
Uçtan uca throughput = min(R1, R2, ..., RN)
Hangi bağlantının kapasitesi en düşükse o tüm aktarımı sınırlar.

1.5 — Protokol Katmanları
Internet Protokol Yığını (5 Katmanlı Model)

Layered Architecture (Katmanlı Mimari) — Karmaşık sistemleri yönetmek için her katman kendi işini yapar ve alttaki katmana hizmet sunar. Modülerlik: bir katman değiştirildiğinde diğerleri etkilenmez.

5 Katmanlı Internet Protokol Yığını & Encapsulation
5. Uygulama (Application) — HTTP, FTP, SMTP, DNS, BitTorrent. Mesaj (message) üretir.
4. Taşıma (Transport) — TCP (güvenilir), UDP (hızlı). Segment oluşturur. Uç-uç iletişim.
3. Ağ (Network) — IP protokolü. Datagram oluşturur. Router'dan router'a yönlendirme.
2. Bağlantı (Link) — Ethernet, WiFi, PPP. Frame oluşturur. Düğümden düğüme iletim.
1. Fiziksel (Physical) — Bitler. Kablo, fiber, radyo üzerinden elektrik/ışık/radyo sinyali.

Encapsulation (Kapsülleme) — Gönderici her katmanda üst katmandan gelen PDU'ya başlık bilgisi ekler. Alıcıda ise her katman kendi başlığını söker ve üst katmana iletir:

Uygulama mesajı (M)
↓ + Transport başlığı (Ht) → Segment [Ht | M]
↓ + Network başlığı (Hn) → Datagram [Hn | Ht | M]
↓ + Link başlığı (Hl) → Frame [Hl | Hn | Ht | M]
↓ Fiziksel bitler olarak iletilir

OSI 7 Katman Modeli — ISO tarafından tanımlanan 7 katmanlı model: Physical, Data Link, Network, Transport, Session, Presentation, Application. İnternet modeli session ve presentation'ı uygulama katmanına dahil eder.

1.6 — Ağ Güvenliği
Temel Saldırı Türleri

Malware (Zararlı Yazılım) — Cihazlara sızan yazılımlar. Botnet oluşturmak için kullanılır. Virüs (insan etkileşimi gerektirir) ve worm (kendi kendine yayılır) şeklinde ikiye ayrılır.

DoS (Denial-of-Service) — Hedef ağı/sunucuyu kullanılamaz hale getirme saldırısı. DDoS (Distributed DoS): binlerce bot aynı anda saldırır.

Packet Sniffing — Paylaşımlı ortamlarda (WiFi, HFC) tüm paketlerin ele geçirilmesi. Wireshark gibi araçlarla yapılabilir. Şifreleme ile önlenir.

IP Spoofing — Sahte kaynak IP adresiyle paket göndermek. Alıcı paketi meşru bir kaynaktan geliyormuş gibi algılar.

Man-in-the-Middle — İki taraf arasındaki iletişime müdahale etme. TLS/SSL ile önlenir.

2.1 — Uygulama Katmanı Temelleri
Ağ Uygulama Mimarileri & Temel Kavramlar

Client-Server Mimarisi — Her zaman açık olan bir server (kalıcı IP, veri merkezi) ve servis talep eden client'lar (dinamik IP olabilir, birbiriyle doğrudan iletişim kurmaz). Örn: Web, FTP, e-posta.

P2P (Peer-to-Peer) Mimarisi — Her zaman açık merkezi sunucu yok. Uygulama zaman zaman bağlanan cihazlar (peer'lar) arasında çalışır. Her peer hem istemci hem sunucu. Kendi kendini ölçekleyebilir (self-scalability) ancak yönetimi zordur. Örn: BitTorrent.

Process — Bir end system üzerinde çalışan program. İstemci process (iletişimi başlatan) ve sunucu process (bağlantıyı bekleyen). Aynı hosttaki process'ler OS'un inter-process communication mekanizmasıyla konuşur.

Socket — Process ile ağ arasındaki yazılımsal arabirim. Uygulamanın ağa mesaj gönderip aldığı "kapı". Transport katmanı API'si.

IP Adresi + Port Numarası — Hedef process'i tanımlamak için ikisi birlikte gerekir. İyi bilinen portlar: HTTP=80, HTTPS=443, SMTP=25, DNS=53, FTP=21.

Transport Hizmet Gereksinimleri:

Güvenilir Veri İletimi (Reliable Data Transfer): TCP garantiler, UDP garantilemez
Throughput: Bant genişliği garantisi (bazı uygulamalar için kritik)
Zamanlama (Timing): Düşük gecikme (gerçek zamanlı uygulamalar)
Güvenlik: Şifreleme, kimlik doğrulama

2.2 — Web ve HTTP
HyperText Transfer Protocol — Detaylı İnceleme

HTTP (HyperText Transfer Protocol) — Web'in uygulama katmanı protokolü. RFC 1945 (HTTP/1.0), RFC 7230 (HTTP/1.1), RFC 7540 (HTTP/2). Stateless (durumsuz): sunucu istemcinin önceki isteklerini hatırlamaz.

Web Page — HTML dosyası ve referans verilen nesnelerden (JPEG, JS, CSS, video) oluşur. Her nesnenin bir URL'i vardır.

Non-Persistent HTTP — Her nesne için ayrı TCP bağlantısı kurulur. Sorun: Her bağlantı için 2 RTT (biri TCP handshake, biri HTTP isteği) + iletim gecikmesi.

Persistent HTTP — Tek TCP bağlantısı üzerinden birden fazla nesne gönderilir. HTTP/1.1 varsayılan olarak persistent bağlantı kullanır. Pipelining ile birden fazla istek sıralı gönderilebilir.

Non-Persistent HTTP Yanıt Süresi:
2RTT + dosya iletim süresi
(1 RTT: TCP bağlantısı, 1 RTT: HTTP istek/yanıt)

[veri]
── Durum kodları: 200 OK, 301 Moved, 304 Not Modified, 404 Not Found, 505 Version Not Supported

Cookie — HTTP stateless olduğu için durum takibi için kullanılır. 4 bileşen: (1) response Set-cookie başlığı, (2) request cookie başlığı, (3) kullanıcının tarayıcısındaki cookie dosyası, (4) sunucudaki veritabanı.

Web Cache (Proxy Server) — İstemci adına kaynak sunucunun HTTP isteklerini karşılayan ağ varlığı. Avantajlar: istemci yanıt süresi azalır, ISP bağlantısındaki trafik azalır. Conditional GET ile tazelik kontrol edilir.

HTTP/2 (RFC 7540, 2015) — HOL (Head-of-Line) blocking sorununu çözer. Request/response multiplexing: tek TCP bağlantısı üzerinde paralel istekler. Server push. Header compression (HPACK).

HTTP/3 — TCP yerine QUIC (UDP tabanlı) kullanır. Google tarafından geliştirildi. Daha düşük kurulum gecikmesi (0-RTT).

2.3 — E-posta
SMTP, POP3, IMAP

E-posta Mimarisinin 3 Bileşeni: User agent (mail istemci: Outlook, Gmail), mail server (posta kutusu ve message queue), SMTP protokolü.

SMTP (Simple Mail Transfer Protocol) — E-posta gönderme protokolü. TCP 25 numaralı port. Push protokolü: gönderici sunucudan alıcı sunucuya iter. RFC 5321. 7-bit ASCII komutlar (HELO, MAIL FROM, RCPT TO, DATA, QUIT).

SMTP vs HTTP Karşılaştırması: Her ikisi de kalıcı bağlantı (persistent connection) kullanır. HTTP pull (istek üzerine çeker), SMTP push (itmece). HTTP her nesneyi ayrı response'a koyar; SMTP tüm mesajı tek gönderide gönderir.

Mail Access Protokolleri:

POP3 (Post Office Protocol v3) — TCP 110
3 aşama: Authorization, Transaction, Update
Mesajları indir ve sil (veya bırak) — durumsuz

IMAP (Internet Mail Access Protocol) — TCP 143
Mesajlar sunucuda tutulur, klasörlerle yönetilir
Durumlu, çok cihazlı erişime uygundur

Web-based E-mail (Gmail, Hotmail)
HTTP ile user agent ↔ mail server arası
2.4 — DNS
İnternetin Dizin Sistemi — Domain Name System

DNS (Domain Name System) — Hostname'leri IP adreslerine çeviren servis. (1) Dağıtık hiyerarşik veritabanı, (2) Uygulama katmanı protokolü (UDP/TCP port 53). RFC 1034, RFC 1035.

DNS Hizmetleri: Hostname-to-IP çevirisi, Host aliasing (canonical isim + takma isimler), Mail server aliasing, Load distribution (yük dağıtımı — bir hostname'e birden fazla IP adresi).

DNS Hiyerarşisi (3 Seviye):

DNS Hiyerarşik Sunucu Yapısı
Root DNS Sunucuları — 13 mantıksal sunucu (A'dan M'ye), yüzlerce fiziksel konum. TLD sunucularının adreslerini bilir.
TLD (Top-Level Domain) Sunucuları — .com, .org, .net, .edu, .gov gibi uzantılar ve .tr, .uk, .fr gibi ülke kodları. Authoritative sunucuların adreslerini bilir.
Authoritative DNS Sunucuları — Kuruluşun hostname-to-IP kayıtlarını tutar. Her organizasyon kendi sunucusunu barındırabilir.
Local DNS Sunucusu — ISP veya kurumun cache'leyen ara DNS sunucusu. İstemcinin DNS sorgularını yönlendirir.

Recursive vs Iterative Sorgu:

Recursive: Local DNS → Root → TLD → Authoritative (her biri bir sonrakine sorar)
Iterative: Sorgulanan sunucu cevabı bilmiyorsa "şuna sor" der, local DNS ilerler
Pratikte: Local DNS recursive, diğerleri iterative kullanır

DNS Caching — DNS sunucuları öğrendikleri kayıtları cache'ler (TTL süresi boyunca). Root sunucular nadiren sorgulanır. Güncelleme gecikmesi sorun olabilir.

DNS Resource Records (RR):

Format: (Name, Value, Type, TTL)
Type A → Name=hostname, Value=IP address
Type NS → Name=domain, Value=auth. hostname
Type CNAME → Name=alias, Value=canonical hostname
Type MX → Name=domain, Value=mail server hostname

DNS Mesaj Formatı: 12-byte header (ID, flags, # sorular, # cevaplar, # authority, # additional). Questions, Answers, Authority, Additional bölümleri.

2.5 — P2P Dosya Dağıtımı
BitTorrent Protokolü

P2P Ölçekleme Avantajı — Client-server'da dağıtım süresi N büyüdükçe lineer artar. P2P'de her yeni peer sisteme bant genişliği ekler.

Client-Server min süre: max(NF/us, F/dmin)
P2P min süre: max(F/us, F/dmin, NF/(us+Σui))
[F: dosya boyutu, us: sunucu upload, dmin: en düşük peer download]

BitTorrent — Dosya eşit boyutlu chunk'lara (256KB) bölünür. Torrent: aynı dosyayı paylaşan peer grubu. Tracker sunucusu aktif peer listesi tutar.

Rarest First — Peer, torrentta en nadir bulunan chunk'ı önce ister. Kolektif olarak dengeyi sağlar.

Tit-for-Tat (Choking) — Peer'lar kendilerine en hızlı veri gönderen 4 komşuya unchoke (gönderim izni) verir. 30 saniyede bir rastgele 1 ek peer unchoke edilir (optimistic unchoking). Bedava bindirmeyi (free riding) önler.

DHT (Distributed Hash Table) — Tracker'sız P2P sistemi. Her peer küçük bir bölümden sorumludur. Dairesel DHT tasarımında komşular bilinir.

2.6 — Video Streaming ve CDN
Netflix, YouTube — Nasıl Çalışır?

Video — Sabit hızda görüntü dizisi. Görüntüler 8bit/piksel ile kodlanır. MPEG, H.264, H.265 gibi codec'lerle sıkıştırılır. Bit rate: 100kbps (düşük kalite) → 4Mbps (HD) → 10Mbps (4K).

DASH (Dynamic Adaptive Streaming over HTTP) — Video birden fazla kalite seviyesinde encode edilir. İstemci bant genişliğine göre chunk'ları farklı kalitede ister. Manifest dosyası chunk URL'lerini içerir. İstemci akıllı, sunucu aptal.

CDN (Content Distribution Network) — Videolar coğrafi olarak dağıtılmış sunuculara (PoP) yerleştirilir. İstemci en yakın CDN node'undan içerik alır. Netflix, Google gibi büyük şirketler kendi CDN'lerini işletir.

Enter Deep vs Bring Home — Enter Deep: CDN sunucularını erişim ISP'lerine kadar sokma (Akamai tarzı). Bring Home: ISP IXP'lerinde az sayıda büyük cluster kurma (Netflix tarzı). CDN seçimi genellikle DNS yönlendirmesiyle yapılır.

3.1–3.3 — Taşıma Katmanına Giriş
UDP, Multiplexing/Demultiplexing

Transport Layer (Taşıma Katmanı) — Ağ üzerindeki iki end system'deki uygulamalar arası mantıksal iletişimi sağlar. Network layer host-to-host, transport layer ise process-to-process iletişim sağlar.

Multiplexing — Gönderici transport katmanı birden fazla socket'ten veri toplar, her segmente header ekler. Hangi uygulamaya gideceği bilgisi encode edilir.

Demultiplexing — Alıcı transport katmanı gelen segmentlerin başlıklarını inceleyip doğru socket'e yönlendirir. Port numarası temel tanımlayıcıdır.

UDP Demux: Hedef IP + hedef port numarası yeterli
TCP Demux: (kaynak IP, kaynak port, hedef IP, hedef port) 4-tuple — her bağlantı için ayrı socket

UDP (User Datagram Protocol) — RFC 768. Bağlantısız (connectionless), güvenilmez. Handshake yok. Başlık sadece 4 alan:

UDP Segment Başlığı (8 byte):
| Kaynak Port (16bit) | Hedef Port (16bit) |
| Uzunluk (16bit) | Checksum (16bit) |
| Uygulama Verisi (Payload) |

UDP Checksum — 1'in tamamlayıcısı yöntemiyle hata tespiti. Pseudo-header (IP src/dst + length) dahil edilir. Hata tespit eder ama düzeltmez.

UDP'nin Kullanım Nedenleri: (1) Bağlantı kurma gecikmesi yok, (2) Bağlantı durumu tutulmaz → daha az kaynak, (3) Küçük header, (4) Tıkanıklık kontrolü yok → istenen hızda gönderim. DNS, SNMP, gerçek zamanlı medya UDP kullanır. Uygulama katmanında güvenilirlik eklenebilinir (QUIC gibi).

3.4 — Güvenilir Veri İletimi Prensipleri
rdt Protokolleri — Adım Adım Tasarım

Güvenilir Veri İletimi — Alt katman (unreliable channel) üzerinde güvenilir iletim sağlama sorunu. TCP'nin temel görevi. Sonlu durum makinesi (FSM) ile modellenir.

rdt1.0: Mükemmel kanal — hata/kayıp yok. Basit gönder/al.

rdt2.0: Bit hatası olabilir. ARQ (Automatic Repeat reQuest):
- Error detection: checksum
- Feedback: ACK (acknowledgement) ve NAK (negative ACK)
- Retransmission (yeniden iletim)
SORUN: ACK/NAK bozulabilir → belirsizlik

rdt2.1: Sequence number eklendi (0, 1). Duplicate paket tanınır.

rdt2.2: NAK yerine en son doğru alınan pakete ACK. NAK-free protokol.

rdt3.0: Bit hatası + paket kaybı olabilir.
- Timer (countdown timer) eklendi
- Timeout → yeniden iletim
- Stop-and-wait: Verimsiz!

Stop-and-Wait Verimsizliği:

Kullanım oranı (Utilization) U = (L/R) / (RTT + L/R)
Örnek: L=1500byte, R=1Gbps, RTT=30ms
U = 0.000012 / 0.030012 ≈ 0.00027 = 0.027%
→ Kanalın %99.97'si boşa gider!

Pipelining — Birden fazla paket ACK beklenmeden gönderilebilir. Window size (N) ile kaç paketin "uçuşta" (in-flight) olabileceği sınırlanır.

Pipelined Kullanım: U = (N × L/R) / (RTT + L/R)
N = window size, N=3 ile U 3 kat artar

Go-Back-N (GBN) — Gönderici N pakete kadar ACK beklemeden gönderir. Alıcı sadece cumulative ACK gönderir. Hata veya timeout durumunda N paketten itibaren tümü yeniden gönderilir (window başından).

Selective Repeat (SR) — Alıcı her paketi bağımsız ACK'lar ve buffer'lar. Sadece kayıp/hatalı paket yeniden gönderilir. Daha verimli ama alıcı buffer'ı gerektirir.

GBN vs SR:
GBN: Basit alıcı, tekrar iletim fazla → küçük N için iyi
SR: Karmaşık alıcı (buffer gerekli), verimli → büyük hata oranı veya büyük window için iyi
SR window boyutu ≤ sıra numarası uzayının yarısı (k-bit seq: ≤ 2^(k-1))
3.5 — TCP
Connection-Oriented Reliable Transport Protocol

TCP — RFC 793, RFC 1122, RFC 5681. Bağlantı-odaklı (connection-oriented), güvenilir, sıralı, çift yönlü (full-duplex), tek noktadan tek noktaya (point-to-point).

TCP Bağlantı Kurma — 3-Way Handshake:

1. İstemci → Sunucu: SYN segment (SYN=1, seq=client_isn)
2. Sunucu → İstemci: SYNACK segment (SYN=1, ACK=1, seq=server_isn, ack=client_isn+1)
3. İstemci → Sunucu: ACK segment (SYN=0, ack=server_isn+1) + veri

TCP Bağlantı Kapatma — 4-Way: FIN → ACK → FIN → ACK. TIME_WAIT durumu 2×MSL (Maximum Segment Lifetime) sürer.

TCP Segment Yapısı:

| Kaynak Port (16) | Hedef Port (16) |
| Sequence Number (32) |
| Acknowledgement Number (32) |
| Header Len | Flags | Receive Window (16) |
| Checksum (16) | Urgent Data Pointer (16) |
| Options (değişken) |
| Veri |

Flags: URG, ACK, PSH, RST, SYN, FIN

Sequence Number — Segment verisinin ilk byte'ının veri akışındaki byte numarası. Başlangıç değeri (ISN) rastgele seçilir.

Acknowledgement Number — Alıcının bir sonraki beklediği byte'ın numarası. Cumulative acknowledgement (birikimli): 542 ACK → 0-541 arası tamam.

Receive Window (rwnd) — Alıcının receive buffer'ında kalan boş alan. Akış kontrolü için kullanılır.

TCP Round Trip Time ve Timeout — Timeout değeri çok kısa olursa gereksiz yeniden iletim; çok uzun olursa kayıp paket tepkisi gecikmeli.

SampleRTT: Gönderilen segment ile ACK arasındaki zaman
EstimatedRTT = (1-α)×EstimatedRTT + α×SampleRTT [α = 0.125]
DevRTT = (1-β)×DevRTT + β×|SampleRTT - EstimatedRTT| [β = 0.25]
TimeoutInterval = EstimatedRTT + 4×DevRTT

TCP Güvenilir Veri İletimi: Pipeline, cumulative ACK, timer, fast retransmit (3 duplicate ACK → hemen yeniden ilet, timeout beklemeden).

Akış Kontrolü (Flow Control) — Gönderici alıcının buffer'ını taşırmaz. rwnd (receive window) değeri segment başlığında bildirilir.

LastByteSent - LastByteAcked ≤ rwnd
3.6–3.7 — TCP Tıkanıklık Kontrolü
Congestion Control — Slow Start, AIMD, CUBIC

Tıkanıklık (Congestion) — Ağa çok fazla kaynak (paket) girince gecikme ve kayıp artar. Tıkanıklık kontrolsüz olunca: gereksiz yeniden iletim → daha fazla yük → "tıkanıklık çöküşü" (congestion collapse).

Congestion Window (cwnd) — Gönderici tarafında tutulan tıkanıklık penceresi. Ağa ne kadar veri "uçuşta" olabileceğini sınırlar.

LastByteSent - LastByteAcked ≤ min{cwnd, rwnd}
Gönderim hızı ≈ cwnd / RTT byte/saniye

Tıkanıklık Tespiti: (1) Timeout: ciddi tıkanıklık, (2) 3 duplicate ACK: hafif tıkanıklık (hızlı yeniden iletim).

Slow Start (Yavaş Başlangıç) — Başlangıçta cwnd = 1 MSS. Her ACK için cwnd += 1 MSS → cwnd üstel büyür. ssthresh'e ulaşınca Congestion Avoidance moduna geçilir.

Congestion Avoidance — cwnd her RTT'de 1 MSS artar (lineer büyüme — AIMD'nin additive increase kısmı).

AIMD (Additive Increase, Multiplicative Decrease):

Kayıp (3 dup ACK) tespit edilince:
ssthresh = cwnd/2
cwnd = ssthresh + 3 MSS (fast recovery)

Timeout tespit edilince:
ssthresh = cwnd/2
cwnd = 1 MSS (Slow Start'a dön)
TCP Congestion Control Durum Makinesi
SLOW START
cwnd = 1 MSS
Her ACK'ta cwnd += 1
Üstel büyüme
cwnd < ssthresh
CONG. AVOIDANCE
Her RTT'de cwnd += 1
Lineer büyüme
cwnd ≥ ssthresh
"AIMD" bölgesi
FAST RECOVERY
3 dup ACK sonrası
cwnd = ssthresh + 3
Her dup ACK'ta +1
Yeni ACK → CA mod

TCP Fairness (Adillik) — K TCP bağlantısı darboğaz bağlantıyı R/K'ya yakınsayarak adil paylaşır. UDP TCP'yi kesebilir (no congestion control). Paralel TCP bağlantıları adilliği bozabilir.

CUBIC TCP — Linux varsayılan. Kayıp sonrası cwnd'yi daha hızlı iyileştirir (kübik fonksiyon). Yüksek bant genişliği-gecikme ürünü (BDP) ortamlar için daha verimli.

Explicit Congestion Notification (ECN) — Network-assisted tıkanıklık kontrolü. Router'lar IP/TCP header'larına ECN bitleri set eder. Paket kaybı olmadan tıkanıklık bildirimi.

4.1 — Ağ Katmanına Giriş
Data Plane vs Control Plane — Forwarding vs Routing

Data Plane (Veri Düzlemi) — Her router'da ayrı ayrı çalışır. Datagram'ı giriş portuna gelince hangi çıkış portuna yönlendirileceğine karar verir. Forwarding (yerel karar): nanosaniye-mikrosaniye hız.

Control Plane (Kontrol Düzlemi) — Tüm ağ üzerinde koordineli çalışır. Router forwarding tablolarını nasıl dolduracağını belirler. Routing (ağ geneli karar): milisaniye-dakika hız.

İki yaklaşım:
Per-router control plane: Her router routing algoritması çalıştırır (OSPF, BGP)
SDN (Software-Defined Networking): Uzak controller forwarding tablolarını hesaplar ve ağa iter

Network Service Models:

Best-Effort (İnternet modeli): Teslim garantisi yok
Paket sırası garantisi yok
Minimum throughput garantisi yok
Gecikme garantisi yok

ATM CBR: Sabit bit hızı, devre benzeri
ATM ABR: Kullanılabilir bit hızı
4.2 — Router'ın İçi
Giriş Portları, Switching Fabric, Çıkış Portları

Router Mimarisi 4 Bileşeni:

Router İç Mimarisi
Giriş Portları (Input Ports) — Fiziksel bağlantı sonlandırma (PHY), link-layer işleme, forwarding tablosunda arama (lookup), line speed'de çalışır.
Switching Fabric — Giriş portlarını çıkış portlarına bağlar. 3 tipi: memory-based, bus-based, crossbar. Crossbar en hızlı.
Çıkış Portları (Output Ports) — Switching fabric'ten gelen paketleri buffer'lar ve fiziksel bağlantıya iletir. Kuyruk (scheduling) yönetimi burada.
Routing Processor — Routing protokollerini çalıştırır, forwarding tablosunu hesaplar, network yönetimi.

Destination-Based Forwarding — En uzun önek eşleşmesi (Longest Prefix Match): forwarding tablosunda paketin hedef IP'si ile en uzun eşleşen ön eke göre çıkış portu seçilir. TCAM (Ternary Content-Addressable Memory) ile donanım hızında arama.

Longest Prefix Match Örneği:
Tablo: 11001000 00010111 00010 → Interface 0
11001000 00010111 00011000 → Interface 1
11001000 00010111 00011 → Interface 2
diğer → Interface 3

Adres 11001000 00010111 00011000 10101010 → Interface 1

HOL (Head-of-Line) Blocking — Giriş portunda kuyrukta bekleyen bir paket, çıkış portu müsait olsa bile önündeki paket engellediğinde yaşanır. Sanal çıkış kuyruğu (VOQ) ile çözülür.

Paket Zamanlama (Packet Scheduling):

FIFO (First In First Out): Sırayla işle, taşmada tail-drop
Priority Queuing: Yüksek öncelikli paketler önce gönderilir
WFQ (Weighted Fair Queuing): Her akışa ağırlıklı bant genişliği
Round Robin: Her akışa sırayla hizmet
4.3 — IP Protokolü
IPv4, Adresleme, DHCP, NAT, IPv6

IPv4 Datagram Formatı:

| Ver (4) | IHL (4) | TOS/DSCP (8) | Total Length (16) |
| Identification (16) | Flags (3) | Fragment Offset (13) |
| TTL (8) | Protocol (8) | Header Checksum (16) |
| Source IP Address (32) |
| Destination IP Address (32) |
| Options (değişken, genellikle boş) |
| Veri (data) |

Önemli alanlar:
TTL: Her router 1 azaltır, 0'a gelince paket düşürülür
Protocol: 6=TCP, 17=UDP, 1=ICMP
IHL (min 20 byte = 160 bit)

IP Fragmentation — MTU (Maximum Transmission Unit): bir link'in taşıyabileceği max datagram boyutu (Ethernet=1500 byte). MTU'yu aşan datagram parçalara bölünür, hedefte yeniden birleştirilir. IPv6'da fragmentation router'da yapılmaz.

IPv4 Adresleme — 32-bit adres. Dotted-decimal notation: 192.168.1.1. Subnetting: host bölümünün bir kısmı subnet için kullanılır.

CIDR (Classless Inter-Domain Routing) — a.b.c.d/x: x bitleri network prefix. Örn: 200.23.16.0/20. Aggregation (route summarization) ile BGP tablolarını küçültür.

Alt ağ (Subnet) hesabı:
IP: 192.168.1.0/24
Subnet mask: 255.255.255.0
Host sayısı: 2^8 - 2 = 254 (ağ adresi ve broadcast çıkarılır)

CIDR prefix /23: 2^9 = 512 host (2 çıkarılır → 510)

DHCP (Dynamic Host Configuration Protocol) — Plug-and-play: host ağa bağlandığında otomatik IP alır. UDP üzerinden çalışır (server port 67, client port 68). 4 adım: DISCOVER → OFFER → REQUEST → ACK. IP adresi kiralanır (lease), süresi dolunca yenilenir.

NAT (Network Address Translation) — Özel IP adresleri (10.x.x.x, 192.168.x.x, 172.16-31.x.x) harici ağda geçersiz. NAT router tek genel IP adresiyle tüm yerel ağın internete çıkmasını sağlar. NAT translation table: (private IP, port) ↔ (public IP, new port). IPv4 adres tükenmesine karşı ara çözüm. Bazı P2P uygulamalar için sorun oluşturur.

IPv6 Neden Gerekli?
IPv4 adresleri 2011'de bitti. IPv6 = 128-bit adres
2^128 ≈ 3.4 × 10^38 adres

IPv6 Datagram:
- Sabit 40-byte başlık (seçenekler "extension headers"
- Fragmentation desteği kaldırıldı
- Checksum kaldırıldı (alt katmana bırakıldı)
- Flow label alanı eklendi

IPv4→IPv6 Geçiş: Tunneling (IPv6 datagramı IPv4 datagramına gömme)
4.4 — Genelleştirilmiş Yönlendirme ve SDN
OpenFlow ve Match-Plus-Action

Generalized Forwarding — Sadece hedef IP'ye göre değil, paket başlığının birden fazla alanına göre eylem alma. "Match-plus-action" paradigması.

Flow Table — Her satır bir akış kuralı: header fields (eşleşme kriterleri) + counters + actions.

OpenFlow Match Fields (11 alan):
Giriş portu, Ethernet src/dst, VLAN ID,
IP src/dst, IP protokol, TCP/UDP src/dst port

Actions:
Forward (porta ilet), Drop (düşür),
Modify header, Send to controller

SDN (Software-Defined Networking) — Data plane (switching donanımı) ile control plane (yazılım) fiziksel olarak ayrılır. Merkezi SDN controller OpenFlow protokolü ile switch'leri programlar. Ağ üzerinde merkezi görünürlük ve programlanabilirlik.

Middleboxes — Ağda router dışında işlev gören cihazlar: NAT kutuları, güvenlik duvarları (firewalls), load balancer'lar, Web önbellek sunucuları.

5.1–5.2 — Yönlendirme Algoritmaları
Link-State (Dijkstra) vs Distance-Vector (Bellman-Ford)

Routing Algorithm — Kaynak-hedef arası "iyi" yol (en düşük maliyetli) belirleme. Ağ = Graf G=(N,E). N: router seti, E: link seti. c(x,y): link maliyeti (bant genişliği tersi, gecikme vb.).

Link-State (LS) Algoritması — Her router tüm ağın tam topolojisini bilir. Link-state broadcast ile tüm maliyetler yayılır. Dijkstra algoritması en kısa yolu hesaplar.

Dijkstra Algoritması:
N': ziyaret edilmiş node seti, D(v): başlangıç→v en kısa yol

Başlatma: D(u) = c(x,u) (komşu) veya ∞ (değil)
Döngü (|N| kez):
1. D(u) en düşük olan w'yu N''e ekle
2. Her komşu v için:
D(v) = min(D(v), D(w) + c(w,v))

Karmaşıklık: O(n²) — öncelik kuyruğuyla O(n log n)

Distance-Vector (DV) Algoritması — Dağıtık, yinelemeli, asenkron. Her router sadece komşularına ulaşım maliyetini bilir. Bellman-Ford denklemi:

Bellman-Ford: dx(y) = minv{c(x,v) + dv(y)}
x: kaynak, y: hedef, v: x'in komşuları

DV Güncellemesi:
dx değişince komşulara gönder
Komşular güncellerse hesapla
Yakınsayana kadar tekrarla

Count-to-Infinity Problemi — DV'de negatif haberler yavaş yayılır. Bir link koparsa ilgili yol maliyeti çok yavaş artar (sonsuz sayıyor gibi). Poisoned Reverse kısmen çözer: "Senden öğrendiğim rotayı sana ∞ maliyetle rapor et."

LS vs DV Karşılaştırması:
LS: Mesaj karmaşıklığı O(n²), hızlı yakınsama, tıkanıklık yok
DV: Mesaj karmaşıklığı düşük, yavaş yakınsama, count-to-infinity
5.3 — Intra-AS Routing
OSPF — Open Shortest Path First

Autonomous System (AS) — Aynı yönetim altındaki router grupları. İnternet AS'lerden oluşur. Her AS'ye benzersiz ASN (Autonomous System Number) atanır.

Intra-AS Routing — AS içi yönlendirme: IGP (Interior Gateway Protocol). Örnekler: OSPF, RIP, IS-IS.

OSPF (Open Shortest Path First) — RFC 2328. Link-state protokolü. Tüm link maliyetleri broadcast edilir (flooding). Her router Dijkstra çalıştırır. IP doğrudan üzerinde çalışır (protokol numarası 89).

OSPF Özellikleri:
• Güvenlik: Yönlendirme mesajları kimlik doğrulamayla korunur
• Çoklu eş maliyetli yol: Birden fazla yol eşit maliyetliyse hepsini kullanır (load balancing)
• Unicast ve Multicast desteği
• Hiyerarşik OSPF: Area'lar ile büyük AS'lerde ölçeklenir
- Backbone area (Area 0): Tüm alanları birbirine bağlar
- Area border routers: Kendi area'larını özetler
5.4 — Inter-AS Routing
BGP — Border Gateway Protocol

BGP (Border Gateway Protocol) — RFC 4271. İnternetin "yapıştırıcısı". Tüm AS'ler aynı BGP protokolünü çalıştırır. İki tipi: eBGP (farklı AS'ler arası), iBGP (aynı AS içinde). Bellman-Ford tabanlı DV protokolü ama path-vector kullanır.

BGP Reklamı (Advertisement) — Prefix (CIDR bloğu) + attributes şeklinde rota reklamı. Önemli attribute'lar:

AS-PATH: Prefixi reklameden AS'lerin listesi — döngü tespiti için
NEXT-HOP: Sonraki hop router'ın IP adresi
LOCAL-PREF: AS içi tercih değeri (yüksek → tercih edilir)
MED (Multi-Exit Discriminator): Komşu AS'e tercih bildirimi

BGP Rota Seçimi — Aynı prefixi reklameden birden fazla yol varsa: (1) LOCAL-PREF en yüksek, (2) AS-PATH en kısa, (3) NEXT-HOP en yakın (hot-potato routing), (4) BGP identifier kriteri.

Hot-Potato Routing — AS içindeki trafik en kısa yolla AS sınırına gönderilir. AS dışındaki maliyeti görmezden gelir. İSP'nin bant genişliği giderini azaltır.

İnter-AS vs İntra-AS Farkı — Intra-AS: yalnızca performans. Inter-AS: politika kritiktir. Hangi trafiği taşır, hangi ASler üzerinden? ISP karlılık, ticari sözleşmeler.

5.5 — SDN Kontrol Düzlemi
SDN Mimarisi & OpenFlow

SDN Kontrolörü 3 Katmanı:

1. İletişim Katmanı (Communication Layer):
Kontrolör ↔ switch arası OpenFlow protokolü

2. Ağ Geneli Durum Yönetimi (Network-wide state management):
Link durumu, host bilgisi, flow tabloları — kontrolörde merkezi veritabanı

3. Ağ Kontrol Uygulamaları (Network-control applications):
Yönlendirme, erişim kontrolü, load balancing — kontrolör üzerinde çalışır

OpenFlow Protokolü — Kontrolör ile OpenFlow switch'i arasında TCP üzerinden çalışır. Mesaj tipleri: kontrolör→switch (features, modify-state, packet-out), switch→kontrolör (packet-in, flow-removed, port-status), simetrik (hello, echo-request/reply).

SDN Örnek Senaryo — S1 switch'i bilmediği bir paketi packet-in ile kontrolöre gönderir. Kontrolör OpenDaylight/ONOS gibi uygulama katmanından öğrendiği topoloji üzerinden yol hesaplar ve S1, S2, S4'e flow table girişleri gönderir (flow-mod). Sonraki paketler kontrolörü atlatarak doğrudan iletilir.

ONOS / OpenDaylight — Açık kaynak SDN kontrolör platformları. Paralel işleme, yüksek erişilebilirlik, Northbound API (uygulama geliştiricilere), Southbound API (OpenFlow, NETCONF).

5.6 — ICMP
Internet Control Message Protocol

ICMP — RFC 792. Host ve router'lar arasında ağ-katmanı bilgisi iletmek için kullanılır. IP datagramı olarak taşınır (protocol=1). Hata bildirimi ve ağ teşhisi.

Önemli ICMP mesaj türleri:
Type 0, Code 0: Echo reply (ping yanıtı)
Type 3, Code 0: Destination network unreachable
Type 3, Code 3: Destination port unreachable
Type 8, Code 0: Echo request (ping isteği)
Type 11, Code 0: TTL expired (traceroute)
Type 12, Code 0: Bad IP header

Traceroute — Serisi TTL=1,2,3... olan UDP paketleri gönderilir. Her router'da TTL tükenenince "TTL expired" ICMP mesajı geri döner. Böylece yol haritası çıkarılır. Son hedef "Destination port unreachable" gönderir.

ICMPv6 — IPv6 için yeni ICMP. Neighbor Discovery Protocol (NDP) işlevlerini de içerir (ARP'ın yerini alır).

5.7 — Ağ Yönetimi
SNMP ve NETCONF/YANG

Ağ Yönetiminin 5 Bileşeni: (1) Yönetim sunucusu (manager), (2) Yönetilen cihazlar (managed devices: router, switch, host), (3) Yönetim protokolleri, (4) Yönetim bilgi tabanı (MIB), (5) Ağ yöneticisi (insan).

SNMP (Simple Network Management Protocol) — RFC 3416. UDP üzerinde çalışır (port 161/162). GET-REQUEST, GET-NEXT-REQUEST, GET-BULK-REQUEST, SET-REQUEST, INFORM-REQUEST, TRAP mesajları. MIB (Management Information Base): yönetilen cihaz bilgilerini tutan yapı.

NETCONF / YANG — Daha modern alternatif. NETCONF: XML tabanlı, TCP/SSH güvenli iletişim. YANG: veri modelleme dili. CLI ve SNMP'nin yerini almakta.

Kaynak: Kurose J.F. & Ross K.W., Computer Networking: A Top-Down Approach, 8th Edition, Pearson, 2020. Chapter notları kitabın orijinal içeriğinden hazırlanmıştır.
